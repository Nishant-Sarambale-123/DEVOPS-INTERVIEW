# ğŸŸ¦ **5. Mutability vs Immutability in ECR tags**

**Mutable Tags**

* Same tag can be overwritten with a new image
* Example: `latest`
* Risk: Deployments may pull different images with same tag

**Immutable Tags**

* Once pushed, tag cannot be changed
* Ensures consistent deployments
* Good for production

# ğŸŸ¦ **9. Why use Secret instead of ConfigMap?**

* ConfigMaps store data **in plain text** â†’ visible to everyone
* Secrets are **Base64 encoded by default**
* Secrets can be encrypted using **encryption at rest**
* Secret access is controlled via **RBAC**
* Secrets are **not logged** by kubelet
* Mounted as tmpfs â†’ not written to disk

â¡ï¸ ConfigMaps = Non-sensitive data
â¡ï¸ Secrets = Sensitive data (passwords, API keys, TLS certs)

# ğŸŸ¦ **10. How to enable encryption for Secret?**

Edit API server with encryption config:

### Step 1: Create file

```yaml
apiVersion: apiserver.config.k8s.io/v1
kind: EncryptionConfiguration
resources:
  - resources:
      - secrets
    providers:
      - aescbc:
          keys:
            - name: key1
              secret: <base64-encoded-32-byte-key>
      - identity: {}
```

### Step 2: Add flag in kube-apiserver:

```
--encryption-provider-config=/etc/kubernetes/encryption-config.yaml
```

### Step 3: Restart API server

### Step 4: Re-encrypt secrets:

```
kubectl get secrets -A -o yaml | kubectl apply -f -
```
